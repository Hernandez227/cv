Tables section

You can create table with the following tags:

  • "< tr > < / tr >":
    		•  It is to include a row into your table
  • "< td >Text that you want to write < /td >":
    		• It is to include columns into your tables

    You can put it subtitles in the tables with the following tag:

    Tags Functionality
  • < thead > < /thead >
    		•  Is to put a subtitle into your table
  • < tbody>All the body code < /body >
    		•  It is to contain all the body section of the table, it is a good practice
  • < tfoot > < tfoot >
    		•  It is the final of the table, the foot
  • < th >Subtitle that you want < /th >
    		•  It is for add a subtitle, it automatically bolds it
  • < tr > < /tr >
    		• It is to add a row into your table but it is for subtitles
  • < td >Text that you want to write < /td >
    		• It is to include columns into your tables

    In the tag "< table > < / table >", you can add different attributes like the following:

    Table attribute Functionality
  • border = "Pixels that you want"
    		•  It is to add a border in your table
  • cellspacing = "Pixels that you want"
    		•  It is to add spaces in pixels between your columns

    Forms section

    There are a lot ways to make a form, lest see the sintaxis first:


    < form class = "" action = "index.html" method = "post"

    After that, you need to write a label tag to set a title for the form with the following tag: < label > Name of your label < /label >

    If you want a square where you can input different information, you need the following tag: < input type = "attribute" name = " " "value = " ">

    In the type attribute, you can change it with the following attributes:

    Attribute Functionality
  • Text
    		•  It is just for text
  • Password
    		•  It is for marked the text and you can input your password
  • Color
    		•  It is for select a color
  • Checkbox
    		•  It is for put a checkbox
  • Submit
    		•  It is just for submit the data into a server
  • File
    		•  It is for upload a file

    You can write a lot of type attributes for the input tag, check the form documentation
